# CLAUDE.md — import_tiles

Утилиты для оффлайн-карты в `rescue-dashboard`. Сейчас здесь один скрипт —
`download_tiles.py`, качает растровые тайлы OSM в `/var/lib/mesh-net/tiles/`.

## Использование

```bash
# Краснодар, 50×50 км, zoom 5-14 (~1500 тайлов, ~25 мин)
# Низкие зумы (5-9) — обзор, чтобы карта не была чёрной при дальнем масштабе.
python3 scripts/import_tiles/download_tiles.py \
    --bbox 38.7,44.8,39.4,45.3 \
    --zoom 5-14
```

bbox = `min_lon,min_lat,max_lon,max_lat` (так в QGIS, geojson и пр.)

Скрипт идемпотентный — повторный запуск докачает только то, что не скачалось.

## Куда лягут файлы

`/var/lib/mesh-net/tiles/{z}/{x}/{y}.png` — рядом с БД, на том же диске.
rescue-api при старте монтирует этот каталог на `/tiles`, дашборд
запрашивает `/tiles/{z}/{x}/{y}.png` — без интернета.

Если каталога нет / он пустой — карта будет серой, маркеры всё равно
рисуются.

## Объёмы

| Регион | Zoom | Тайлы | МБ | Время (rate=1s) |
|---|---|---|---|---|
| Город (50×50 км) | 10–14 | ~1.5 K | ~30 | ~25 мин |
| Город (50×50 км) | 10–16 | ~25 K | ~500 | ~7 ч |
| Регион (200×200 км) | 10–14 | ~25 K | ~500 | ~7 ч |
| Россия | 8–10 | ~10 K | ~200 | ~3 ч |

Для горно-туристических троп оптимум **z14–16**: на z14 видны улицы,
на z15–16 — отдельные здания и тропы.

## OSM Tile Usage Policy — серая зона

Формально tile.openstreetmap.org **запрещает** автоматическую массовую
загрузку. На практике для образовательных проектов с разумным
rate-limit'ом (1–2 req/sec) и `User-Agent`, идентифицирующим проект,
банов почти не бывает.

Скрипт уже настроен правильно:
- `User-Agent: MeshNetTropy/0.1 (educational ...)`
- `rate=1.0s` между запросами
- остановка при HTTP 429 / 403

Если OSM начнёт банить — увеличь `--rate` до 2–3 сек.
