# Протокол LoRa Mesh v1

## Радиоуровень

| Параметр | Значение |
|----------|----------|
| Частота | 868 МГц (EU ISM) |
| Spreading Factor | SF10 |
| Bandwidth | 125 кГц |
| Coding Rate | 4/5 |
| Мощность TX | +22 дБм (максимум SX1262) |
| Преамбула | 8 символов |
| CRC | включён |
| Дальность (прямая видимость) | до 10 км |
| Дальность (горный рельеф) | 2–5 км |
| Скорость передачи данных | ~980 бит/с при SF10/BW125 |
| Время передачи пакета (64 байта) | ~660 мс |

## Структура пакета (64 байта, фиксированная)

```
 0       1       2       3       4       5       6       7
 +-------+-------+-------+-------+-------+-------+-------+-------+
 |version| type  |    device_id  |channel|  ttl  |      lat      |
 +-------+-------+-------+-------+-------+-------+-------+-------+
 |      lat      |      lon      |      lon      |   timestamp   |
 +-------+-------+-------+-------+-------+-------+-------+-------+
 |   timestamp   |               payload (44 байта)              |
 +-------+-------+               ...                             +
 |                                                               |
 +                               ...             +-------+-------+
 |                                               |    crc16      |
 +-----------------------------------------------+-------+-------+
```

### Поля

| Поле | Смещение | Размер | Тип | Описание |
|------|---------|--------|-----|----------|
| version | 0 | 1 | uint8 | Версия протокола. Текущая: `1` |
| type | 1 | 1 | uint8 | Тип пакета (см. ниже) |
| device_id | 2 | 2 | uint16 LE | Уникальный ID устройства |
| channel | 4 | 1 | uint8 | `0`=TOURIST, `1`=RESCUE |
| ttl | 5 | 1 | uint8 | Time-to-live, 3–5. Декрементируется при ретрансляции |
| lat | 6 | 4 | int32 LE | Широта × 1 000 000 (градусы × 1e6) |
| lon | 10 | 4 | int32 LE | Долгота × 1 000 000 |
| timestamp | 14 | 4 | uint32 LE | Unix timestamp (UTC) |
| payload | 18 | 44 | bytes | Данные пакета (тип зависит от `type`) |
| crc | 62 | 2 | uint16 LE | CRC-16/CCITT-FALSE по байтам 0–61 |

### Типы пакетов

| Код | Имя | Payload | Описание |
|-----|-----|---------|----------|
| 0x00 | PING | пусто | Периодическое сообщение о местоположении (каждые 60 с) |
| 0x01 | CHAT | UTF-8 строка (44 байта, null-terminated) | Текстовое сообщение |
| 0x02 | SOS | UTF-8 причина (44 байта) или пусто | Сигнал бедствия |
| 0x03 | ACK | 2 байта: подтверждаемый device_id | Подтверждение получения SOS |

## Логика Mesh (Flooding)

### Ретрансляция

Каждый узел при получении пакета:

1. Проверяет CRC — если неверный, дропает
2. Проверяет cache дедупликации по ключу `(device_id, timestamp)` — если видел, дропает
3. Добавляет в кэш дедупликации (LRU, 50 записей)
4. Декрементирует `ttl`
5. Если `ttl == 0` — не ретранслирует, только передаёт в локальный обработчик
6. Если `ttl > 0` — ставит в очередь на передачу

### Очередь передачи

- SOS-пакеты идут **вне очереди** (высший приоритет)
- PING-пакеты могут быть вытеснены при переполнении очереди (размер очереди: 10 пакетов)
- Перед отправкой — случайная задержка 0–200 мс (избегаем коллизий)

### SOS-повтор

Отправляющий узел посылает SOS-пакет **3 раза** с интервалом 500 мс.
Дедупликация по `(device_id, timestamp)` — ретрансляторы пропускают дубли.

---

## Каналы

### TOURIST (channel=0)
- Принимают **все** узлы
- Трекинг, чат, SOS туристов

### RESCUE (channel=1)
- Принимают только узлы с `device_id` в whitelist (`RESCUE_WHITELIST` env var)
- Служебные сообщения спасателей
- ACK-пакеты для подтверждения SOS

## Примеры пакетов

### PING от устройства #0x0047

```
01 00 47 00 00 03 A2 C4 8E 02 5B 8E A8 03 67 E5 AD C0
^  ^  ^     ^  ^  ^           ^           ^
|  |  |     |  |  lat(43.355) lon(42.514) ts
|  |  |     |  ttl=3
|  |  |     channel=TOURIST
|  |  device_id=0x0047
|  type=PING
version=1
[44 байта payload = нули]
[2 байта CRC]
```

### SOS от устройства #0x0047

```python
import struct, time

lat = int(43.355123 * 1e6)  # 43355123
lon = int(42.514567 * 1e6)  # 42514567
ts  = int(time.time())
payload = b"Trauma leg, need help\x00" + b'\x00' * 22  # 44 bytes

header = struct.pack('<BBHBBiiI', 1, 2, 0x0047, 0, 3, lat, lon, ts)
packet = header + payload
crc = crc16(packet)
packet += struct.pack('<H', crc)
# отправляем 3 раза с интервалом 500 мс
```

## CRC

Алгоритм: **CRC-16/CCITT-FALSE**  
Poly: `0x1021`, Init: `0xFFFF`, RefIn: `false`, RefOut: `false`, XorOut: `0x0000`

```python
def crc16(data: bytes) -> int:
    crc = 0xFFFF
    for b in data:
        crc ^= b << 8
        for _ in range(8):
            if crc & 0x8000:
                crc = (crc << 1) ^ 0x1021
            else:
                crc <<= 1
        crc &= 0xFFFF
    return crc
```

```cpp
// C++ для ESP32
uint16_t crc16(const uint8_t* data, size_t len) {
    uint16_t crc = 0xFFFF;
    for (size_t i = 0; i < len; i++) {
        crc ^= (uint16_t)data[i] << 8;
        for (int j = 0; j < 8; j++) {
            crc = (crc & 0x8000) ? (crc << 1) ^ 0x1021 : crc << 1;
        }
    }
    return crc;
}
```

## Версионирование

При изменении структуры пакета — инкрементировать `version`.  
Узлы с несовместимой версией дропают пакет (не ретранслируют).  
Текущая версия: **1**.
